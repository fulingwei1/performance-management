## 🔧 常用命令

### 服务管理

```bash
# 启动所有服务
docker-compose up -d

# 停止所有服务
docker-compose down

# 重启服务
docker-compose restart

# 查看状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
docker-compose logs -f frontend
docker-compose logs -f mysql
```

### 数据库操作

```bash
# 进入MySQL容器
docker exec -it ate_mysql bash

# 连接数据库
docker exec -it ate_mysql mysql -uroot -pperformance123 performance_db

# 备份数据库
docker exec ate_mysql mysqldump -uroot -pperformance123 performance_db > backup.sql

# 恢复数据库
docker exec -i ate_mysql mysql -uroot -pperformance123 performance_db < backup.sql

# 查看表列表
docker exec -it ate_mysql mysql -uroot -pperformance123 -e "USE performance_db; SHOW TABLES;"

# 查看员工数据
docker exec -it ate_mysql mysql -uroot -pperformance123 -e "USE performance_db; SELECT id, name, department, role FROM employees LIMIT 5;"
```

### 重新构建

```bash
# 重新构建所有镜像
docker-compose build --no-cache

# 重新构建特定服务
docker-compose build --no-cache backend

# 构建并启动
docker-compose up -d --build
```

## 🐛 故障排查

### 问题1: 端口冲突

**现象**: 启动失败，提示端口已被占用

**解决**:
```bash
# 查看占用的进程
lsof -i :3306
lsof -i :3001
lsof -i :5173

# 杀死进程
kill -9 <PID>

# 或修改docker-compose.yml中的端口映射
```

### 问题2: MySQL启动失败

**现象**: Backend无法连接数据库

**解决**:
```bash
# 查看MySQL日志
docker-compose logs mysql

# 删除数据卷重新初始化
docker-compose down -v
docker-compose up -d
```

### 问题3: Backend构建失败

**现象**: TypeScript编译错误

**解决**:
```bash
# 本地测试构建
cd backend
npm install
npm run build

# 查看错误后修复,然后重新构建
docker-compose build --no-cache backend
```

### 问题4: 数据初始化慢

**现象**: 首次启动员工数据加载慢

**原因**: 173名员工需要逐一bcrypt加密密码

**解决**: 等待2-3分钟，查看backend日志确认完成

```bash
docker-compose logs -f backend | grep "成功初始化"
```

## 🔒 生产环境安全建议

⚠️ **当前配置仅适用于本地测试！**

生产环境部署前务必:

1. **修改所有默认密码**:
   - MySQL root密码
   - MySQL用户密码
   - JWT_SECRET
   - ADMIN_DEFAULT_PASSWORD

2. **限制MySQL端口暴露**:
   ```yaml
   # docker-compose.yml中注释掉
   # ports:
   #   - "3306:3306"
   ```

3. **启用HTTPS**:
   - 配置Nginx SSL证书
   - 使用Let's Encrypt

4. **添加反向代理**:
   - 使用Nginx/Caddy作为反向代理
   - 隐藏后端端口

5. **数据备份**:
   - 定时备份MySQL数据
   - 备份上传到云存储

```yaml

  command:
    - --max_connections=200
    - --innodb_buffer_pool_size=1G

    - --slow_query_log=1
    - --long_query_time=2
```
