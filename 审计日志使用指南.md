# 审计日志使用指南

**功能**: 记录系统中所有用户操作，支持查询、过滤、统计分析  
**权限**: HR 和 admin 可查看所有日志，普通用户只能查看自己的日志  
**版本**: v1.0  
**更新时间**: 2026-02-13

---

## 📋 功能概述

审计日志自动记录系统中的所有重要操作，包括：

- ✅ 用户登录/登出
- ✅ 创建记录（工作总结、晋升申请、申诉等）
- ✅ 修改记录（评分、审批、目标调整等）
- ✅ 删除记录
- ✅ 审批操作

每条日志包含：
- 操作人信息（ID、姓名、角色）
- 操作详情（动作、模块、目标）
- 请求信息（IP、浏览器、URL）
- 操作结果（成功/失败）
- 时间戳

## 🔍 如何查询日志

### 1. 查询所有日志（HR/admin）

**接口**: `GET /api/audit-logs`

**支持的筛选参数**:
- `user_id`: 用户ID
- `action`: 操作类型（CREATE/UPDATE/DELETE/LOGIN/LOGOUT）
- `module`: 模块（auth/performance/promotion/appeal/objective等）
- `target_type`: 目标类型（performance_record/promotion_request等）
- `result`: 结果（SUCCESS/FAILED/UNAUTHORIZED）
- `start_date`: 开始日期（YYYY-MM-DD）
- `end_date`: 结束日期（YYYY-MM-DD）
- `page`: 页码（默认1）
- `limit`: 每页数量（默认50）

**示例**:

```bash
# 查询所有登录记录
curl -X GET "http://localhost:3001/api/audit-logs?action=LOGIN" \
  -H "Authorization: Bearer {token}"

# 查询某用户的所有操作
curl -X GET "http://localhost:3001/api/audit-logs?user_id=e001" \
  -H "Authorization: Bearer {token}"

# 查询某时间段的失败操作
curl -X GET "http://localhost:3001/api/audit-logs?result=FAILED&start_date=2026-02-01&end_date=2026-02-13" \
  -H "Authorization: Bearer {token}"

# 组合筛选：查询某用户在绩效模块的所有操作
curl -X GET "http://localhost:3001/api/audit-logs?user_id=e001&module=performance&page=1&limit=20" \
  -H "Authorization: Bearer {token}"
```

### 2. 查询用户操作历史

**接口**: `GET /api/audit-logs/user/:userId`

**参数**:
- `limit`: 查询数量（默认100）

**权限**: 本人或HR/admin

**示例**:

```bash
# 查询自己的操作历史
curl -X GET "http://localhost:3001/api/audit-logs/user/e001?limit=50" \
  -H "Authorization: Bearer {token}"
```

**用途**:
- 员工查看自己的操作记录
- HR 审计某员工的所有操作
- 追踪问题时查看操作时间线

### 3. 查询对象操作历史

**接口**: `GET /api/audit-logs/target/:type/:id`

**权限**: 仅HR/admin

**示例**:

```bash
# 查询某条工作总结的所有操作
curl -X GET "http://localhost:3001/api/audit-logs/target/performance_record/rec-e001-2026-02" \
  -H "Authorization: Bearer {token}"

# 查询某个晋升申请的操作历史
curl -X GET "http://localhost:3001/api/audit-logs/target/promotion_request/pr-12345" \
  -H "Authorization: Bearer {token}"
```

**用途**:
- 查看某条记录被谁修改过
- 追踪数据变更历史
- 审计敏感操作

### 4. 查询统计信息

**接口**: `GET /api/audit-logs/stats`

**权限**: 仅HR/admin

**示例**:


## 💡 常见使用场景

### 场景1: 追踪数据变更

**问题**: 某条工作总结的分数被修改了，需要知道谁改的？

**解决方案**:

```bash
# 1. 查询该记录的操作历史
curl -X GET "http://localhost:3001/api/audit-logs/target/performance_record/rec-e001-2026-02" \
  -H "Authorization: Bearer {token}"

# 2. 查看返回的日志，找到 action=UPDATE 的记录
# 3. 查看 user_name 和 created_at，确认修改人和时间
```

### 场景2: 审计员工操作

**问题**: 需要审计某员工在系统中的所有操作

**解决方案**:

```bash
# 查询该员工的所有操作
curl -X GET "http://localhost:3001/api/audit-logs/user/e001" \
  -H "Authorization: Bearer {token}"
```

### 场景3: 排查系统问题

**问题**: 系统中出现了批量删除操作，需要排查

**解决方案**:

```bash
# 查询所有删除操作
curl -X GET "http://localhost:3001/api/audit-logs?action=DELETE" \
  -H "Authorization: Bearer {token}"
```

### 场景4: 监控登录安全

**问题**: 检查是否有异常登录

**解决方案**:

```bash
# 查询所有登录失败记录
curl -X GET "http://localhost:3001/api/audit-logs?action=LOGIN&result=FAILED" \
  -H "Authorization: Bearer {token}"

# 查询某用户的登录历史
curl -X GET "http://localhost:3001/api/audit-logs?user_id=admin&action=LOGIN" \
  -H "Authorization: Bearer {token}"
```

### 场景5: 生成审计报告

**问题**: 需要生成本月的系统使用报告

**解决方案**:

```bash
# 1. 查询统计信息
curl -X GET "http://localhost:3001/api/audit-logs/stats" \
  -H "Authorization: Bearer {token}"

# 2. 查询本月的所有操作
curl -X GET "http://localhost:3001/api/audit-logs?start_date=2026-02-01&end_date=2026-02-28" \
  -H "Authorization: Bearer {token}"
```

## 🗄️ 直接查询数据库

如果有数据库访问权限，也可以直接查询：

```sql
-- 1. 查看最近的100条日志
SELECT 
  user_id, user_name, action, module, description, 
  result, created_at
FROM audit_logs 
ORDER BY created_at DESC 
LIMIT 100;

-- 2. 查询某用户的操作
SELECT * 
FROM audit_logs 
WHERE user_id = 'e001' 
ORDER BY created_at DESC;

-- 3. 查询失败的操作
SELECT 
  user_id, user_name, action, module, 
  error_message, created_at
FROM audit_logs 
WHERE result = 'FAILED' 
ORDER BY created_at DESC;

-- 4. 统计每个模块的操作数
SELECT 
  module, 
  COUNT(*) as total,
  SUM(CASE WHEN result = 'SUCCESS' THEN 1 ELSE 0 END) as success,
  SUM(CASE WHEN result = 'FAILED' THEN 1 ELSE 0 END) as failed
FROM audit_logs 
GROUP BY module 
ORDER BY total DESC;

-- 5. 查询某对象的操作历史
SELECT 
  user_id, user_name, action, description, 
  changes, created_at
FROM audit_logs 
WHERE target_type = 'performance_record' 
  AND target_id = 'rec-e001-2026-02'
ORDER BY created_at DESC;

-- 6. 查询今天的所有操作
SELECT * 
FROM audit_logs 
WHERE created_at >= CURRENT_DATE 
ORDER BY created_at DESC;

-- 7. 查询某IP的所有操作
SELECT * 
FROM audit_logs 
WHERE ip_address = '192.168.1.100' 
ORDER BY created_at DESC;
```

## 📝 日志字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | integer | 日志ID（自增）|
| user_id | string | 操作人ID |
| user_name | string | 操作人姓名 |
| user_role | string | 操作人角色 |
| action | enum | 操作类型（CREATE/UPDATE/DELETE/LOGIN/LOGOUT）|
| module | string | 模块（auth/performance/promotion/appeal等）|
| target_type | string | 目标类型（performance_record/promotion_request等）|
| target_id | string | 目标ID |
| description | text | 操作描述（中文）|
| changes | jsonb | 修改内容（JSON格式）|
| ip_address | string | 操作IP地址 |
| user_agent | text | 浏览器信息 |
| request_method | string | HTTP方法（POST/PUT/DELETE）|
| request_url | text | 请求URL |
| result | enum | 操作结果（SUCCESS/FAILED/UNAUTHORIZED）|
| error_message | text | 失败原因 |
| created_at | timestamp | 操作时间 |

## ⚠️ 注意事项

1. **性能考虑**
   - 日志查询支持分页，建议不要一次查询太多数据
   - 默认每页50条，最多建议不超过200条
   - 使用筛选条件可以提升查询效率

2. **数据保留**
   - 日志会永久保存（当前版本）
   - 未来可能会实现自动归档（超过6个月）

3. **隐私保护**
   - 审计日志包含敏感信息，仅限HR和admin查看
   - 不要将日志内容泄露给无关人员

4. **异常情况**
   - 如果审计日志写入失败，不会影响业务操作
   - 系统会在控制台记录错误，但不会抛出异常

## 🚀 快速开始

### 测试审计日志功能

```bash
# 1. 登录获取token
TOKEN=$(curl -s -X POST http://localhost:3001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}' | jq -r '.data.token')

# 2. 执行一些操作（会被记录）
curl -X POST http://localhost:3001/api/performance/summary \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"month":"2027-02","summary":"测试","achievements":"测试","issues":"无"}'

# 3. 查询审计日志
curl http://localhost:3001/api/audit-logs -H "Authorization: Bearer $TOKEN" | jq .

# 4. 查询统计信息
curl http://localhost:3001/api/audit-logs/stats -H "Authorization: Bearer $TOKEN" | jq .
```
