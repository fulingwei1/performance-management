## 📚 文档内容概览

### 《任职资格标准.md》

#### 结构框架
```
1. 职级体系概述
   - 9个职级定义（L1-L6, M1-M3）
   - 晋升基本要求

2. 通用能力标准
   - 绩效要求（量化标准）
   - 工作年限要求
   - 通用能力要求

3. 分岗位标准（6个岗位）
   - 测试工程师（L2-L6，5个职级）
   - 开发工程师（L2-L4，3个职级）
   - 产品经理（L2-L3，2个职级）
   - 项目经理（L3，1个职级）
   - 销售岗位（L2-L3，2个职级）
   - 管理岗位（M1-M2，2个职级）

4. 晋升评估清单
   - 员工自评清单
   - 审批人评估清单

5. 附录
   - 涨薪参考比例
   - 认证推荐
   - 学习资源
```

#### 核心亮点

**1. 量化标准**
- ✅ 绩效：明确分数和等级要求
- ✅ 年限：明确最低和建议年限
- ✅ 技能：明确必备技能数量（如 8/12）

**示例：L3 中级测试工程师**
```
- 工作年限：2-5年（或 L2 满 1.5 年）
- 绩效要求：近6个月平均 ≥ 70分（B级）
- 技能要求：必备 8/12 项
  1. ✅ 熟练掌握测试用例设计方法
  2. ✅ 掌握至少 1 种自动化测试工具
  3. ✅ 能编写基本的自动化测试脚本
  ... (共12项，必备8项)
```

**2. 结构化描述**
- 基本要求（年限/绩效/薪资）
- 技能要求（必备/优选）
- 工作能力
- 典型产出
- 晋升要点

**3. 全面覆盖**
- 技术岗位：测试、开发
- 产品岗位：产品经理、项目经理
- 业务岗位：销售
- 管理岗位：经理、总监

### 《晋升审批-对照清单.md》

#### 审批流程（5步）

**第一步：硬性条件检查**
- 工作年限
- 绩效要求
- 违规记录

**第二步：技能要求检查**
- 技能清单对照表
- 逐项核对
- 统计达标数量

**第三步：工作能力评估**
- 独立工作能力
- 问题解决能力
- 团队协作能力
- 学习成长能力

**第四步：工作成果评估**
- 工作量
- 工作质量
- 突出成果

**第五步：晋升合理性评估**
- 涨薪比例合理性
- 与同级对比
- 晋升后价值

#### 综合评分

**评分表：**
| 评估项 | 权重 | 评分 | 得分 |
|--------|------|------|------|
| 工作年限 | 10% | 1-5 | ____ |
| 绩效表现 | 30% | 1-5 | ____ |
| 技能掌握 | 25% | 1-5 | ____ |
| 工作能力 | 20% | 1-5 | ____ |
| 工作成果 | 15% | 1-5 | ____ |
| **总分** | **100%** | | **____** |

**通过标准：**
- 总分 ≥ 4.0：建议通过
- 总分 3.0-3.9：可以通过，但需改进
- 总分 < 3.0：建议拒绝

## 🎯 使用方式

### 员工使用

**申请前：**
1. 查阅《任职资格标准.md》
2. 找到目标职级（如 L3 中级测试工程师）
3. 对照技能清单自评
4. 确认符合要求后再申请

**申请时：**
- 在申请表中突出已掌握的技能
- 提供证明材料（项目/认证/成果）
- 对照标准填写能力总结

---

### 审批人使用

**审批时：**
1. 打开《晋升审批-对照清单.md》
2. 打开员工的晋升申请
3. 打开《任职资格标准.md》找到目标职级
4. 逐项对照评估
5. 填写评分表
6. 做出审批决定

**审批意见模板：**

**✅ 通过示例：**
```
该员工表现优秀，符合L3中级测试工程师要求：
1. 工作年限达标（已工作2.5年）
2. 绩效表现优异（近6个月平均75分，B级）
3. 技能掌握充分（已掌握9/12项必备技能）
4. 工作成果突出（主导自动化框架搭建，覆盖率80%）

建议批准晋升，涨薪15%。
```

**❌ 拒绝示例：**
```
该员工暂不符合L3中级测试工程师要求，主要问题：
1. 技能掌握不足（仅掌握6/12项必备技能，需要8项）
2. 缺少自动化测试经验（L3必备）
3. 绩效刚刚达标（平均70分，没有余量）

建议改进方向：
1. 学习自动化测试工具（Selenium/Playwright）
2. 编写自动化测试脚本（至少50个）
3. 提升绩效到75分以上

建议6个月后再申请。
```

## 💡 优势与局限

### ✅ 优势

**1. 快速上线**
- 无需开发
- 2 小时完成
- 立即可用

**2. 标准明确**
- 量化要求（分数/数量）
- 结构化描述
- 便于对照

**3. 全面覆盖**
- 9 个职级
- 6 个岗位
- 每个职级 10-20 项要求

**4. 易于使用**
- 文档清晰
- 有对照清单
- 有评分表

**5. 灵活调整**
- 文档易修改
- 可根据实际调整
- 快速迭代

---

### ⚠️ 局限

**1. 需要人工对照**
- 审批人需要逐项核对
- 耗时较长（10-15 分钟/人）
- 依赖审批人主观判断

**2. 无自动匹配**
- 系统不能自动对比
- 不能自动标记符合/不符合
- 不能自动计算得分

**3. 数据分散**
- 标准在文档，申请在系统
- 需要两边切换查看
- 容易遗漏

**4. 无差距分析**
- 员工不知道自己差多少
- 不能自动生成改进建议
- 需要审批人手动告知
