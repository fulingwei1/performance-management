## 🔴 问题描述

### 原始问题
用户登录时需要同时提供 `username`、`password` 和 `role` 三个参数：

```typescript
// 原代码
body('role').isIn(['employee', 'manager', 'gm', 'hr', 'admin'])

// 登录请求
{
  "username": "admin",
  "password": "123456",
  "role": "admin"  // ❌ 用户不应该知道自己的系统角色
}
```

### 用户体验问题
- ❌ 用户必须预先知道自己的系统角色
- ❌ 容易选错角色导致登录失败
- ❌ 增加了不必要的认知负担
- ❌ 不符合常规登录流程
