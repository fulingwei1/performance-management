# HR数据导入 - 快速参考卡片

## 🎯 三步完成导入

### 第1步：下载模板 ✏️
- 组织架构模板（CSV格式）
- 员工名册模板（CSV格式）

### 第2步：填写数据 📝

#### 组织架构必填字段：
```
部门编码 | 部门名称 | 上级部门编码 | 部门类型
TEST     | 测试部   | TECH         | subDepartment
```

#### 员工名册必填字段：
```
员工编号 | 姓名 | 部门编码 | 角色    | 职级         | 直属上级
E001     | 张三 | TEST     | employee| intermediate | M001
M001     | 经理 | TEST     | manager | senior       |
```

**考核关系关键**：员工的"直属上级编号"填写经理的员工编号

### 第3步：上传导入 🚀
1. 选择CSV文件
2. 预览数据（系统自动检查错误）
3. 确认导入

---

## 📋 字段速查表

### 角色（Role）
| 值 | 含义 | 考核权限 |
|----|------|----------|
| employee | 普通员工 | 被考核 |
| manager | 部门经理 | 考核下属 |
| gm | 总经理 | 考核经理 |
| hr | 人力资源 | 系统管理 |

### 职级（Level）
| 值 | 中文 | 说明 |
|----|------|------|
| senior | 高级 | 高级工程师/资深员工 |
| intermediate | 中级 | 中级工程师/熟练员工 |
| junior | 初级 | 初级工程师/新员工 |
| assistant | 助理 | 助理/实习生 |

### 部门类型
| 值 | 用途 |
|----|------|
| company | 根节点（总经办）|
| department | 一级部门 |
| subDepartment | 二级部门/小组 |

---

## ✅ 数据检查清单

导入前确认：
- [ ] 部门编码唯一，无重复
- [ ] 员工编号唯一，无重复
- [ ] 所有员工的"部门编码"在组织架构中存在
- [ ] 所有员工的"直属上级编号"是已存在的经理
- [ ] 经理的"直属上级编号"留空
- [ ] 日期格式为 YYYY-MM-DD
- [ ] CSV文件编码为UTF-8（避免中文乱码）

---

## ⚠️ 常见错误及解决

| 错误提示 | 原因 | 解决 |
|----------|------|------|
| 部门编码已存在 | 重复导入 | 如需更新，直接导入；如需新建，更换编码 |
| 直属上级不存在 | 经理未导入或编号错误 | 先导入经理，检查编号 |
| 部门未定义 | 组织架构未导入 | 先导入组织架构 |
| 角色值不合法 | 拼写错误 | 使用 employee/manager/gm/hr |

---

## 📊 导入顺序（重要！）

```
第一步：组织架构 → 建立部门
     ↓
第二步：员工名册 → 建立考核关系
```

**必须先导入组织架构！** 员工名册依赖部门数据。

## 💡 实用技巧

### 技巧1：Excel编辑，CSV导入
1. 在Excel中编辑数据（方便复制粘贴）
2. 另存为CSV格式（逗号分隔）
3. 上传CSV文件

### 技巧2：分批导入
数据量大时分批：
- 第一批：导入所有经理（建立考核链）
- 第二批：导入各部门员工

### 技巧3：先小批量测试
- 先导入3-5个部门测试
- 验证无误后再导入全部数据

## 📞 遇到问题？

**第一步**：检查CSV格式是否正确
**第二步**：查看系统错误提示，定位问题行
**第三步**：对照本手册的"常见错误"章节
**第四步**：如仍无法解决，导出错误日志联系技术支持
