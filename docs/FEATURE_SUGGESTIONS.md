# 功能优化与补充建议

> 基于当前已实现能力与常见绩效管理需求整理，供产品迭代参考。

---

## 一、角色与入口

### 1.1 总经理（GM）独立工作台

**现状**：GM 登录后跳转到 `/hr/dashboard`，侧边栏有「工作台 / 经理评分 / 数据分析」但对应路由 `/gm/dashboard`、`/gm/scoring`、`/gm/analytics` 未在 `App.tsx` 中注册，GM 点击会落到 404 或默认重定向。

**建议**（二选一或分阶段）：

- **方案 A**：为 GM 增加独立路由，复用或封装经理端能力：
  - `/gm/dashboard`：总经理工作台（下属部门经理列表、待我评分数量、关键指标）
  - `/gm/scoring`：对部门经理的评分（与 HR 的「总经理评分」业务一致，但入口在 GM 侧）
  - `/gm/analytics`：全公司/多部门数据分析
- **方案 B**：GM 登录后直接进入「总经理评分」业务页（例如 `/hr/gm-scoring`），侧栏仅保留 GM 需要的少数菜单，避免出现无效链接。

## 二、考核流程闭环

### 2.1 申诉流程

**现状**：考核周期中有「申诉截止日期」配置，但缺少员工申诉入口和 HR 处理申诉的流程。

**建议**：

- **员工端**：在「我的分数」或独立菜单增加「绩效申诉」：
  - 选择某条已完成的绩效记录，填写申诉理由，在申诉截止日前提交。
- **HR 端**：增加「申诉管理」页：
  - 列表展示待处理/已处理申诉，支持查看申诉内容、关联绩效记录、填写处理结果（维持/调整）、并可选通知员工。

数据层需增加申诉表（关联绩效记录 ID、申诉人、理由、状态、处理人、处理意见、时间等）。

### 2.2 结果公示

**现状**：文档中标注「⏳ 结果公示」，当前无公示或发布结果的入口。

**建议**：

- HR 在考核周期结束后可「发布本周期结果」：
  - 发布后，员工在「我的分数」中可见该周期最终结果（或仅可见等级/总分，按权限控制）。
- 可选：公示名单/公示范围配置（如仅部门内可见、仅本人可见等），与权限配合。

### 2.3 截止日与提醒

**现状**：周期配置中有「提前提醒天数」「超时自动提交」，经理评分页有静态截止日提示，但无主动推送。

**建议**：

- **站内提醒**：
  - 各角色首页或统一「消息/待办」区：如「您有 X 条待提交总结」「Y 人待评分」「距自评截止还有 Z 天」。
- **推送（可选）**：
  - 与企业微信/钉钉/邮件集成，在提醒日发送「绩效总结即将截止」等消息；或先做站内消息，推送后续迭代。
- **自动提交**：
  - 若保留「超时自动提交」，需在后端或定时任务中实现：到自评截止日未提交则按规则生成或标记为自动提交，并与前端状态一致。

---

## 三、员工端

### 3.1 待办与进度

**现状**：员工可写总结、查分数，但缺少与周期强绑定的待办提示。

**建议**：

- 员工工作台或总结页显式展示：
  - 当前考核周期、自评截止日、本月总结是否已提交/已评分；
  - 若未提交，突出「待提交」入口和剩余天数。

### 3.2 历史与对比

**现状**：已有「我的分数」等展示单次评分。

**建议**：

- 支持按周期/月份查看历史得分，并可选简单趋势（如折线图：各月总分或等级），便于员工感知成长与反馈。

---

## 四、经理端

### 4.1 催办与批量操作

**现状**：经理可逐条评分，有截止日提示。

**建议**：

- **催办**：对待提交/待评分的下属，支持「发送催办」（先做站内提醒，后续可接推送）。
- **批量操作**：如「批量标记已阅」「导出本团队本周期评分表」等，按实际使用场景增加。

### 4.2 与考核周期联动

**现状**：经理评分多按「月份」筛选，与 HR 配置的考核周期可能不完全一致。

**建议**：

- 经理端「评分」或「工作台」展示当前生效的考核周期及经理评分截止日；
  - 待办数量、列表筛选与周期状态一致（如仅显示当前周期内的待评分）。

---

## 五、HR 端

### 5.1 申诉管理

见 **2.1 申诉流程**，需在 HR 侧增加申诉处理页面与状态流。

### 5.2 结果发布与公示

见 **2.2 结果公示**，由 HR 触发发布并控制可见范围。

### 5.3 报表与分析

**现状**：已有归一化报告、数据导出、总经理评分等。

**建议**（按需）：

- **多维度分析**：部门对比、同比/环比、等级分布趋势等，便于 HR 做汇报和调优。
- **导出增强**：支持按周期/部门导出带评语和等级的完整报表，便于归档或对接薪酬。

---

## 六、体验与细节

### 6.1 修改密码后会话

**建议**：修改密码成功后，视安全策略决定是「仅提示成功」还是「清除 token 并跳转登录」，避免旧 token 继续使用。

### 6.2 移动端与响应式

**建议**：若需在手机/平板上使用，对关键页（登录、工作总结、经理评分、HR 待办）做响应式或简单移动版，至少保证核心流程可完成。

### 6.3 无障碍与提示

**建议**：关键操作（提交总结、提交评分、发布结果）前二次确认或 Toast 明确成功/失败原因；重要列表提供加载状态与空状态文案。

---

## 七、优先级建议

| 优先级 | 项 | 说明 |
|--------|----|------|
| 高 | GM 入口与路由 | 解决 GM 登录后侧栏无效、体验割裂 |
| 高 | 员工待办/截止日提示 | 与周期联动，减少漏提交流程 |
| 中 | 申诉流程（员工提交 + HR 处理） | 闭环周期中已配置的「申诉截止」 |
| 中 | 站内提醒/待办 | 各角色首页或统一消息区 |
| 中 | 结果发布与公示 | 与「结果公示」需求对齐 |
| 低 | 经理催办与批量操作 | 提升经理效率 |
| 低 | 多维度报表与导出增强 | 支撑分析与归档 |
| 低 | 修改密码后会话策略、移动端 | 体验与安全增强 |

以上可按迭代资源择项实现；先补齐 GM 入口与考核流程闭环（申诉、公示、提醒），再扩展报表与体验细节。
