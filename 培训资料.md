### 第一部分：系统介绍（15 分钟）

#### 1.1 传统绩效管理的痛点

**PPT 要点：**

| 痛点 | 具体表现 |
|------|---------|
| 📋 流程繁琐 | 多个 Excel 表格来回传，容易丢失 |
| ⏰ 时间滞后 | 月底才填总结，很多事情已经忘了 |
| 📊 数据分散 | HR 要手动汇总，容易出错 |
| 🔒 不够透明 | 员工不知道自己被怎么评价的 |
| 🎯 目标脱节 | 个人目标与公司战略脱节 |

**培训师说：**

> 举个例子：小王上个月做了 5 个项目，月底写总结时只记得最近的 2 个，前面 3 个全忘了。经理打分时也只能凭印象，公平吗？
>
> 再比如：HR 要汇总 200 个人的考核表，一个个复制粘贴到 Excel，光这个工作就要 2 天。万一某个经理忘记交，还要一个个去催...
>
> 这些问题，我们的系统都能解决。

#### 1.2 系统核心价值

**PPT 要点：**

```
🎯 战略对齐
   公司战略 → 部门目标 → 个人目标
   层层分解，确保每个人的工作都在为公司目标服务

📝 过程管理
   不是月底才想起来写总结
   随时记录工作进展，月底一键生成

🤖 AI 赋能
   不知道怎么写？AI 帮你写
   15+ 场景智能助手，提升效率

📊 数据驱动
   实时仪表板，一目了然
   领导不用等报表，随时看数据

⚖️ 公平透明
   评分有依据，申诉有渠道
   让绩效管理真正公平
```

#### 1.3 系统功能模块

**演示：打开系统首页**

**培训师说：**

> 系统分为 5 大模块，我们一个个来看：
>
> 1. **目标管理** - 设定目标、审批、跟进度

> 3. **绩效评分** - 上级评分 + 同事互评

##### 场景 1：设定个人目标（8 分钟）

**操作步骤：**

1. **登录系统**
   ```
   工号/姓名：e001 或 姚洪
   密码：123456
   ```

2. **进入目标管理**
   - 点击左侧「我的目标」
   - 点击「+ 新增目标」

3. **填写目标信息**
   ```
   目标类型：OKR
   目标描述：提升测试用例覆盖率，减少线上 Bug 率
   
   关键结果：
   KR1: 自动化测试覆盖率达到 80% (权重 40%)
   KR2: 月均线上 Bug 数 < 3 个 (权重 35%)
   KR3: 完成 5 个核心模块的单元测试 (权重 25%)
   
   截止时间：2026-06-30
   ```

4. **使用 AI 助手优化**
   - 点击「目标描述」右侧的 🤖 图标
   - AI 自动优化成：
     ```
     通过提升测试自动化水平和用例质量，
     将自动化测试覆盖率提升至 80%，
     同时将月均线上 Bug 数控制在 3 个以内，
     确保产品质量稳定性。
     ```
   - 点击「采用」或手动修改

5. **提交审批**
   - 检查信息无误
   - 点击「提交审批」
   - 状态变为「待审批」

**培训师强调：**

> 📌 **SMART 原则**：
> - Specific（具体）：目标要清晰明确
> - Measurable（可衡量）：要有数字指标
> - Achievable（可实现）：不要定不可能完成的目标
> - Relevant（相关）：与部门目标对齐
> - Time-bound（有时限）：设定明确截止时间
>
> AI 助手会帮你检查这些原则，但最终还是要自己判断。

##### 场景 2：填写月度工作总结（10 分钟）

**操作步骤：**

1. **进入工作总结**
   - 点击左侧「工作总结」
   - 在「待完成」标签页找到任务

2. **点击填写**
   - 任务名称：2026年1月工作总结
   - 截止时间：2026-02-03 23:59

3. **填写工作内容**
   - 手动输入或点击 AI 助手

   **手动输入示例：**
   ```
   1. 完成了用户登录模块的测试用例编写
   2. 修复了 15 个测试环境的 Bug
   3. 参与了新项目的需求评审
   ```

   **点击 AI 助手：**
   - 系统自动生成专业表述
   - 可以多次生成，选最满意的版本
   - 也可以在 AI 生成基础上继续修改

4. **填写工作成果**
   ```
   1. 测试用例库新增 50 条用例，覆盖率提升 15%
   2. 发现并协助开发修复 15 个 Bug，提升系统稳定性
   3. 输出《用户登录模块测试报告》，获领导好评
   ```

5. **填写遇到的问题**
   ```
   1. 部分老旧模块缺少文档，编写用例困难
   2. 测试环境不稳定，影响测试效率
   ```

6. **填写下月计划**
   ```
   1. 完成支付模块的测试用例编写
   2. 推动测试环境稳定性优化
   3. 学习性能测试工具 JMeter
   ```

7. **提交审核**
   - 检查内容完整性
   - 点击「提交审核」

**培训师强调：**

> 📌 **写总结的技巧**：
> - 用数据说话：不要写「做了很多」，要写「做了 50 条用例」
> - 突出成果：不是列工作流水账,要写产出了什么价值
> - 问题要具体：不要写「有困难」，要写具体是什么困难
> - 计划要可执行：下月要做的事要具体明确
>
> AI 助手可以帮你润色，但内容要自己想。AI 不知道你具体做了什么。

##### 场景 3：同事互评（7 分钟）

**操作步骤：**

1. **进入同事互评**
   - 点击左侧「同事互评」
   - 查看 HR 分配给你的待评价同事

2. **开始评价**
   - 点击某个同事的「开始评价」
   - 注意：被评价人**看不到**是你评的

3. **逐项打分**
   ```
   工作能力：4 分（良好）
   团队协作：5 分（优秀，经常主动帮助同事）
   沟通能力：4 分（良好）
   创新意识：3 分（合格）
   ```

4. **填写评价意见**
   
   **手动输入：**
   ```
   该同事工作认真负责，专业能力强，
   团队协作意识好，经常主动帮助新人。
   建议在创新方面多尝试新方法。
   ```

   **或使用 AI 助手优化：**
   - AI 会生成更规范的评语
   - 避免口语化表述

5. **提交评价**
   - 提交后不可修改
   - 确认无误后提交

**培训师强调：**

> 📌 **互评原则**：
> - 客观公正：不要因为私人关系影响评价
> - 具体事例：举具体例子，不要空洞
> - 建设性：既要肯定优点，也要提改进建议
> - 匿名保密：不要向被评价人透露
>
> 互评的目的是帮助同事成长，不是打击人。

##### 场景 4：提交申诉（5 分钟）

**操作步骤：**

1. **查看考核成绩**
   - 点击「考核结果」
   - 发现某月评分明显偏低

2. **提交申诉**
   - 点击「绩效申诉」
   - 点击「+ 提交申诉」

3. **填写申诉信息**
   ```
   申诉类型：评分异议
   考核月份：2026年1月
   
   申诉理由：
   1月份我完成了 3 个紧急项目，加班 40+ 小时
   但评分只有 65 分（C级），明显与实际付出不符
   
   我的工作成果：
   - 项目 A 提前 3 天交付
   - 项目 B 测试发现 20+ 个关键 Bug
   - 项目 C 获得客户高度认可
   
   请求重新评估成绩，至少应达到 B 级（70分以上）
   ```

4. **使用 AI 优化表述**
   - 让申诉理由更有说服力
   - 规范语言表达

5. **提交申诉**
   - HR 会在 5 个工作日内处理

**培训师强调：**

> 📌 **申诉建议**：
> - 提交前先与直属经理沟通，很多问题可以直接解决
> - 申诉要有理有据，提供具体证据
> - 态度要诚恳，不要情绪化
> - 不要频繁申诉，会影响印象

   **手动输入示例：**

   ```
   - 2025年12月：88分（A级）
   ```

   ```
   1. 掌握自动化测试框架 Selenium、Playwright
   ```

   ```
   1. 能独立完成复杂项目测试
   2. 主导过3个大型项目测试方案
   3. 协助经理培训新员工
   4. 积极参与部门技术分享
   ```

6. **填写工作总结**
   ```
   1. 2025年完成15个项目测试
   2. 发现200+个缺陷，发现率95%
   3. 主导自动化框架开发，提效40%
   4. 获"年度优秀员工"称号
   5. 零客户投诉记录
   ```

7. **提交申请**
   - 检查所有信息
   - 点击「提交申请」
   - 等待经理审批

**培训师强调：**

> 📌 **晋升申请要点**：
> - 充分准备：提前整理工作成果
> - 用数据说话：具体的数字和案例
> - 合理期望：涨薪比例要符合实际
> - 突出重点：重点展示核心成果
>
> 晋升审批流程：员工提交 → 经理审批 → 总经理审批 → HR审批 → 通过/拒绝

#### 2.2 经理操作流程（20 分钟）

**培训师准备：**
- 切换账号：`m006` / `123456`（宋魁，经理）

---

##### 场景 1：审批下属目标（5 分钟）

**操作步骤：**

1. **进入目标审批**
   - 点击「目标管理」→「待审批目标」
   - 查看下属提交的目标

2. **评估目标合理性**
   - 目标是否清晰具体？
   - 关键结果可衡量吗？
   - 难度是否合适？
   - 与部门目标对齐吗？

3. **三种操作**
   
   **✅ 通过（目标合理）**
   - 直接点击「通过」
   - 员工收到通知，目标生效

   **❌ 驳回（目标不合理）**
   - 点击「驳回」
   - 填写驳回理由：
     ```
     目标不够具体，"提升质量"太模糊，
     请修改为可量化的指标，如"Bug率降低30%"
     ```
   - 员工收到通知，重新修改

   **✏️ 调整（微调后通过）**
   - 点击「调整」
   - 直接修改目标内容
   - 填写调整理由：
     ```
     时间太紧，从 3 月底调整为 6 月底
     ```
   - 系统自动通过

**培训师强调：**

> 📌 **审批技巧**：
> - 审批前可以先口头沟通，避免反复驳回
> - 驳回理由要具体，告诉员工怎么改
> - 调整要与员工确认，不要擅自修改

---

##### 场景 2：审核工作总结（5 分钟）

**操作步骤：**

1. **进入总结审核**
   - 点击「总结审核」
   - 查看待审核列表

2. **阅读总结内容**
   - 工作量饱和吗？
   - 成果达到预期吗？
   - 问题真实吗？

3. **填写经理评语**
   ```
   工作内容丰富，成果突出，尤其是 XX 项目表现优秀。
   建议在时间管理上进一步优化，避免加班过多。
   下月计划合理，继续保持。
   ```

4. **通过/驳回**
   - 通过：员工收到评语
   - 驳回：员工重新修改

**培训师强调：**

> 📌 **评语原则**：
> - 肯定为主，批评为辅
> - 给出具体建设性建议
> - 不要只写"good"、"不错"这种空话

---

##### 场景 3：绩效评分（5 分钟）

**操作步骤：**

1. **进入绩效评分**
   - 点击「绩效评分」
   - 选择考核月份

2. **选择员工评分**
   - 参考依据：
     - 目标完成度
     - 工作总结
     - 日常观察
     - 同事反馈

3. **逐项打分**
   ```
   工作业绩（40%）：85 分
   工作能力（30%）：80 分
   工作态度（20%）：90 分
   团队协作（10%）：85 分
   
   综合得分：85 分（A级）
   ```

4. **填写总体评价**
   ```
   该员工本月表现优秀，目标完成度高，
   工作态度积极，团队协作好。
   建议继续保持，争取下月冲击 S 级。
   ```

5. **提交评分**
   - ⚠️ 提交后不可修改，请仔细检查

**培训师强调：**

> 📌 **评分注意事项**：
> - 同一标准评估所有下属，避免宽严不一
> - 优秀员工要敢于给高分（90+）
> - 问题员工要如实反映，不要和稀泥
> - 评分后建议面谈沟通，让员工知道哪里做得好、哪里要改进

---

##### 场景 4：审批晋升申请（5 分钟）

**操作步骤：**

1. **进入晋升审批**
   - 点击「晋升加薪」
   - 查看下属的晋升申请

2. **查看申请详情**
   - 当前职级 → 目标职级
   - 期望涨薪比例
   - 绩效总结
   - 技能/能力/工作总结
   - 最近绩效记录

3. **评估申请**
   
   **评估标准：**
   - 绩效是否达标？（最近6个月平均分）
   - 技能是否符合要求？
   - 工作贡献是否突出？
   - 涨薪比例是否合理？

4. **做出决定**

   **✅ 通过申请**
   - 点击「通过」
   - 填写审批意见：
     ```
     该员工表现优秀，技能扎实，完全达到
     中级工程师要求。建议通过晋升，
     涨薪10%合理。
     ```
   - 申请流转到总经理

   **❌ 拒绝申请**
   - 点击「拒绝」
   - 填写拒绝理由：
     ```
     该员工工作表现良好，但技能水平尚未
     达到中级要求。建议再工作6个月，
     提升自动化测试能力后再申请。
     ```
   - 申请结束

**培训师强调：**

> 📌 **审批注意事项**：
> - 客观评估，不要主观判断

### 第三部分：AI 助手使用技巧（15 分钟）

#### 3.1 AI 助手能做什么？

**PPT 要点：**

| 场景 | AI 能帮你 |
|------|-----------|
| 📝 写目标 | 优化目标描述，检查 SMART 原则 |
| 📋 写总结 | 规范表述，突出成果，生成专业语言 |
| 💬 写评语 | 生成客观公正的评价意见 |
| 📢 写申诉 | 规范申诉理由，提升说服力 |
| ✏️ 写计划 | 生成可执行的工作计划 |

#### 3.2 如何让 AI 生成更好的内容？

**培训师演示：**

**❌ 不好的用法：**
```
什么都不输入，直接点 AI 生成
→ AI 生成的是通用模板，没有个性化内容
```

**✅ 正确的用法：**
```
1. 先手动输入要点（关键词即可）
   例如："完成了登录模块测试，发现15个Bug"

2. 点击 AI 助手优化
   → AI 生成："完成用户登录模块的功能测试和回归测试，
        共计编写测试用例 50 条，发现并跟踪 15 个缺陷，
        协助开发团队提升模块稳定性。"

3. 在 AI 生成基础上继续修改
   → 加入具体数据、个人思考
```

#### 3.3 AI 使用注意事项

**PPT 要点：**

⚠️ **AI 不是万能的：**
- AI 不知道你具体做了什么，内容要自己提供
- AI 生成的数据可能不准确，要核实
- AI 的评价可能太官方，要结合实际调整

✅ **正确使用方式：**
- AI 是助手，不是替代品
- AI 帮你润色表达，不是替你编造内容
- AI 生成后要检查修改，不要直接照搬

💡 **最佳实践：**
1. 先自己写关键内容（30%）
2. AI 帮你扩展润色（50%）
3. 自己检查修改完善（20%）

#### Q&A 清单

**Q1: 忘记密码怎么办？**

**Q2: 提交的内容可以修改吗？**
A: 未审批前可撤回修改，已审批后不可改（特殊情况联系经理/HR）。

A: 超期未提交会自动冻结，联系 HR 申请解冻（需说明理由）。

**Q5: 同事互评是匿名的吗？**
A: 是的，被评价人看不到是谁评的。

**Q6: 申诉成功后会怎样？**
A: HR 会要求经理重新评估成绩，如确实不公会调整分数。

**Q7: 可以查看历史记录吗？**
A: 可以，在相应模块选择历史月份即可查看。

**Q8: 系统支持手机吗？**
A: 支持，但建议用电脑操作体验更好（尤其是填写总结）。

**Q9: 数据会丢失吗？**
A: 所有数据实时保存到数据库，不会丢失（建议还是及时提交）。

**Q10: 系统卡顿怎么办？**
A: 刷新页面 / 清除缓存 / 联系技术支持。

### 时间控制
- 每部分开始时告知时长（"这部分大概15分钟"）
- 设置中场休息（1 小时后休息 5-10 分钟）
- 预留 buffer 时间（计划 120 分钟，实际可能 135 分钟）
